Lesson 0008 - 30 minutes

Retrieval evals tell you whether search actually improved.

A RAG system without retrieval evals is mostly vibes. A small golden set gives every retrieval change a scoreboard.

The One Idea

Before changing chunk size, embedding model, filters, hybrid search, or reranking, write down questions and the source chunks that should be retrieved.

The answer generator cannot cite the right evidence if retrieval never finds it. Evaluate retrieval separately before judging the final answer.

Golden Case Shape

{
  "id": "refund-enterprise-001",
  "question": "Can enterprise customers get a refund?",
  "expectedChunkIds": ["refunds:enterprise:30-day"],
  "mustNotReturn": ["refunds:self-serve:trial"],
  "notes": "Should find enterprise policy, not consumer trial policy."
}

What To Track

  • Did the expected chunk appear in the top results?
  • Did irrelevant or forbidden chunks appear?
  • Did filters remove inaccessible evidence?
  • Did the final answer cite retrieved evidence?

Useful First Metrics

MetricQuestion It AnswersGood Early Use
Recall@k Did at least one expected chunk appear in the top k results? Use it to catch missing evidence after chunking or embedding changes.
Precision@k How much of the retrieved set was actually relevant? Use it to reduce noisy context packets.
Forbidden Hit Rate Did blocked or inaccessible chunks appear? Use it to test metadata filters and tenant boundaries.
Citation Hit Rate Did the final answer cite an expected source? Use it to connect retrieval quality to answer behavior.

Interactive Scoring: Pass Or Fail?

Expected chunk is result #2, no forbidden chunks are returned, and the answer cites it.

Expected chunk is missing, but the answer sounds plausible.

Expected chunk appears, but a restricted executive memo also appears in results.

Build Assignment

Write five retrieval eval cases for your tiny corpus.

  • Two direct policy questions.
  • One ambiguous question that should retrieve two chunks.
  • One question with a forbidden chunk that must be filtered out.
  • One question with no sufficient evidence.

Ask the agent to convert the cases into a TypeScript array and propose simple pass/fail checks.
