Lesson 0002 - 25 minutes

An AI feature begins as a model call, but ships as an application boundary.

Today you learn the smallest useful runtime shape: input, instructions, model parameters, response handling, and telemetry.

The One Idea

A model call is just an API call with an unusual property: the service can be creative, wrong, slow, expensive, or format-breaking. Senior AI engineering starts when you make those possibilities explicit in code.

Think of the call as a boundary object: your product sends task context across the boundary, receives a candidate answer, validates it, logs it, and decides what the user should see next.

The Four Runtime Steps

1. Compose

Gather user input, product state, policy, examples, and output requirements. Keep irrelevant context out.

2. Call

Choose the model, parameters, timeout, retry policy, and streaming mode. This is a production dependency.

3. Validate

Check structure, citations, refusal behavior, and domain constraints before trusting the output.

4. Observe

Record request shape, latency, token usage, cost, failures, and user feedback. You cannot improve what you cannot see.

A TypeScript Skeleton

type ProductBriefRequest = {
  featureRequest: string;
  audience: "pm" | "engineering" | "leadership";
};

type ModelCallResult = {
  text: string;
  latencyMs: number;
  model: string;
  usage?: {
    inputTokens: number;
    outputTokens: number;
  };
};

async function createBrief(input: ProductBriefRequest) {
  const startedAt = Date.now();

  const response = await ai.responses.create({
    model: "your-model",
    input: [
      { role: "developer", content: "Create a concise product triage brief." },
      { role: "user", content: input.featureRequest }
    ]
  });

  return {
    text: response.output_text,
    latencyMs: Date.now() - startedAt,
    model: response.model,
    usage: response.usage
  } satisfies ModelCallResult;
}

What To Notice

  • The model call is hidden behind a product function, not scattered through the UI.
  • The response returns metadata, not only text.
  • The output is still plain text. Lesson 0003 fixes that with structured outputs.
  • There is no eval yet. Lesson 0006 or later will add a scenario harness.

Build Assignment

Create a tiny TypeScript module named mentally, not necessarily on disk yet: createProductBrief(input). It should define:

  • The input type.
  • The output type.
  • The model call location.
  • The metadata you want to log: latency, model, token usage, failure reason.

The win is not the API syntax. The win is that the AI dependency now has a contract your app can test.
