Lesson 0019 - 30 minutes

AI observability should not become accidental data hoarding.

Prompts, outputs, retrieved chunks, traces, memories, and eval examples all need retention and redaction rules.

The One Idea

AI systems create rich debugging data. That data is valuable, but it can also contain private user text, customer documents, secrets, tool outputs, or regulated information.

The right design captures enough to debug and evaluate behavior while reducing the amount, sensitivity, and lifetime of stored data.

Minimize

Store IDs, summaries, hashes, and structured outcomes when full raw text is not needed.

Redact

Remove secrets, credentials, payment data, or unnecessary personal information before logging.

Expire

Set retention windows for traces, memories, eval data, caches, and uploaded documents.

Separate

Keep sensitive payloads away from broad analytics, support tools, and cross-tenant dashboards.

Retention Policy Shape

type RetentionRule = {
  artifact: "prompt" | "output" | "trace" | "memory" | "eval_case" | "cache";
  storeRawText: boolean;
  redaction: "none" | "secrets" | "pii" | "tenant_sensitive";
  retentionDays: number;
  accessRole: "user" | "team_admin" | "platform_admin";
};

AI Data To Classify

  • User prompts and uploaded files.
  • Retrieved chunks and citations.
  • Model outputs and structured fields.
  • Tool arguments and tool results.
  • Traces, eval failures, and memories.

Interactive Drill: Store Raw Or Redact?

A trace contains an API key returned inside a tool error.

An eval case stores only chunk IDs, pass/fail score, and non-sensitive failure category.

A support trace includes a customer's private contract text that is not needed for debugging.

Build Assignment

Create a data retention table for your AI portfolio app.

  • List six artifacts: prompts, outputs, traces, memories, evals, caches, files, or tool results.
  • Decide whether each stores raw text.
  • Assign a retention window and access role.
  • Mark one artifact that needs redaction before storage.

Ask the agent to review your table as if it were going into a fintech production design doc.
