Lesson 0017 - 30 minutes

Use LLM judges for judgment, not for everything.

A judge model is useful when quality is semantic, but it should sit beside deterministic checks and human review.

The One Idea

Some AI behavior cannot be scored with a simple string match. A judge model can grade groundedness, helpfulness, completeness, or policy adherence.

But judges are still models. They can be inconsistent, biased by wording, overconfident, or too forgiving. Treat them as measured instruments, not truth itself.

Best Use

Semantic checks: is the answer grounded, does it address the question, does it avoid unsupported claims?

Pair With Code

Use deterministic checks for schema validity, missing citations, blocked tools, and exact policy constraints.

Calibrate

Compare judge scores against human labels and keep examples of false positives and false negatives.

Explain Scores

Ask for a short rationale so failures become actionable instead of mysterious red numbers.

Judge Rubric Shape

{
  "criterion": "groundedness",
  "score": "pass | fail",
  "rubric": [
    "Every factual claim is supported by cited evidence.",
    "The answer refuses when evidence is insufficient.",
    "The answer does not invent policy details."
  ],
  "rationale": "One short paragraph."
}

Guardrails For Judges

  • Give the judge the question, answer, and source evidence.
  • Use narrow rubrics, not vague "is this good?" prompts.
  • Separate independent criteria into separate scores.
  • Sample human review for important decisions.

Interactive Drill: Judge Or Code?

Check whether the output includes a valid riskLevel enum.

Check whether the answer adds policy claims not supported by citations.

Check whether a side-effecting tool call was attempted without approval.

Build Assignment

Add one judge-based scorer and two deterministic scorers to your eval harness.

  • Judge scorer: groundedness, usefulness, or policy adherence.
  • Code scorer: schema validity or required citation presence.
  • Code scorer: blocked tool calls, token budget, or latency budget.
  • Write three examples where the judge might be wrong.

Ask the agent to tighten your judge rubric until it has observable criteria.
