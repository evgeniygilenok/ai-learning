Lesson 0032 - 35 minutes

State is the run ledger the workflow can resume from.

State management records where the agent is, what it has learned during this run, what approvals are pending, and what can safely happen next.

The One Idea

State is not memory. State is the current workflow snapshot: enough structured data to validate transitions, resume after interruption, retry safely, and explain what happened.

Good state management makes agent behavior boring: every step reads state, returns a state patch, and moves through an allowed transition.

Run Identity

Run ID, tenant, user, task type, current owner, and timestamps.

Workflow Position

Current state, allowed next states, step count, retry count, and stop reason.

Known Facts

Facts discovered in this run: ticket details, selected files, tool results, and confidence.

Pending Decisions

Clarifying questions, approval requests, blocked tools, and user decisions.

Budgets

Token, latency, cost, max step, and max retry limits for the run.

Audit Pointers

Trace IDs, tool call IDs, approval IDs, and links to durable logs.

Run State Shape

type AgentRunState = {
  runId: string;
  currentState: "intake" | "clarify" | "plan" | "dry_run" | "review" | "execute" | "done";
  task: { title: string; risk: "low" | "medium" | "high" };
  facts: Record<string, unknown>;
  pendingApprovals: ToolApproval[];
  budgets: { maxSteps: number; stepsUsed: number; maxToolRetries: number };
  traceIds: string[];
};

State Rules

  • Use schemas for state patches.
  • Persist before external side effects.
  • Make interruptions resumable.
  • Keep secrets out of state unless a secure vault owns them.
  • Store references to large artifacts instead of copying everything.

Common Failure

The prompt says "remember what you did," but the app has no durable state. That creates brittle workflows because recovery depends on model recall instead of a validated run snapshot.

Build Assignment

Design the state object for the same Jira-style coding workflow agent:

  • List the fields needed to resume after human approval.
  • List the fields needed to prevent duplicate side effects.
  • List the fields needed for the final summary.
  • Define one state transition that must be rejected by code.

Ask the agent to review your state shape for missing recovery and audit fields.
