Lesson 0025 - 35 minutes

Evaluate agents by the path, not only the final answer.

A workflow can end with plausible text while taking unsafe tools, skipping approvals, or burning too many steps.

The One Idea

Agent evaluation needs trace-based metrics. You score the final result, but also the steps: state transitions, tool choice, approval gates, recovery behavior, latency, and cost.

This is where traces become test fixtures.

Outcome Quality

Did the workflow produce the right plan, checklist, answer, or draft?

Path Validity

Were state transitions allowed, bounded, and recoverable?

Tool Safety

Were tools scoped, authorized, idempotent, and approved when needed?

Operating Cost

Did the agent stay within step, latency, and token budgets?

Trace Case Shape

type AgentEvalCase = {
  task: string;
  expectedStates: string[];
  forbiddenTools: string[];
  requiredApprovals: string[];
  maxSteps: number;
  successRubric: string[];
};

Score The Run

  • Final artifact quality.
  • Correct state sequence.
  • No forbidden tools.
  • Required approvals present.
  • Step and cost budget respected.

Interactive Drill: Which Metric Fails?

The final checklist is good, but the agent used a write tool before approval.

The agent loops between plan and clarify until it hits max steps.

The path is clean, but the final PR checklist omits the main test requirement.

Build Assignment

Create five eval cases for your controlled workflow agent.

  • Two normal success cases.
  • One missing-info case that must clarify.
  • One risky-tool case that must ask for approval.
  • One failed-tool case that must recover or stop cleanly.

Ask the agent to turn the cases into trace assertions.
