Lesson 0023 - 35 minutes

Hybrid search broadens recall. Reranking sharpens precision.

A strong RAG retriever often combines semantic search, keyword search, metadata filters, and a final reranking step.

The One Idea

Vector search is great for meaning. Keyword search is great for exact names, IDs, error codes, and domain terms. Hybrid retrieval combines both, then reranking chooses the best evidence for the context packet.

Vector Candidates

Find semantically similar chunks even when wording differs.

Keyword Candidates

Capture exact product names, ticket IDs, policy labels, and technical terms.

Merge And Filter

Deduplicate, apply ACL filters, and preserve source diversity.

Rerank

Score candidates against the query and keep only the strongest evidence.

Retriever Shape

type RetrievalPlan = {
  vectorTopK: number;
  keywordTopK: number;
  filters: RetrievalFilter;
  rerankTopK: number;
  contextBudgetTokens: number;
};

When Hybrid Helps

  • Queries include acronyms, IDs, or exact product names.
  • Semantic search returns plausible but wrong neighbors.
  • Users mix natural language with domain terms.
  • The context packet is noisy and needs a sharper final pass.

Build Assignment

Upgrade your retrieval design from vector-only to hybrid search.

  • Pick three query types that need keyword matching.
  • Define candidate counts for vector and keyword search.
  • Add dedupe, ACL filtering, and reranking steps.
  • Write two eval cases where hybrid should beat vector-only.

Ask the agent to review whether your hybrid design improves recall without flooding the context packet.
