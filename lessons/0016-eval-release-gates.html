Lesson 0016 - 30 minutes

An eval harness becomes useful when it can block a risky release.

A good release gate turns "the prompt feels better" into a concrete pass, fail, or needs-review decision.

The One Idea

An eval is not just a notebook you run after something breaks. For production-shaped AI work, evals should run before prompt, retrieval, model, or tool-policy changes are accepted.

The gate does not need to be perfect. It needs to protect important behavior, expose regressions, and leave a trail explaining why a change shipped.

Golden Cases

Representative scenarios with expected behavior, source expectations, and known failure modes.

Scorers

Code checks, schema checks, retrieval checks, human labels, or judge models that produce scores.

Thresholds

Minimum pass rates or regression limits that define whether a change can ship.

Review Lane

A manual path for ambiguous failures, new cases, or intentional behavior changes.

Release Gate Shape

type EvalGate = {
  suite: "triage-v1" | "rag-v1" | "workflow-v1";
  changedArtifact: "prompt" | "retriever" | "model" | "tool_policy";
  requiredPassRate: number;
  maxCriticalFailures: number;
  mustPassCaseIds: string[];
  reviewer?: string;
};

What Should Block?

  • Schema validity regresses.
  • Critical safety cases fail.
  • Expected source citations disappear.
  • Unauthorized tool calls are allowed.
  • Latency or cost exceeds the product budget.

Interactive Drill: Ship Or Block?

The new prompt improves tone but fails two high-risk tool approval cases.

Pass rate is unchanged, latency drops 18%, and no must-pass cases fail.

RAG answer quality seems better, but expected citations disappear in refund cases.

Portfolio Signal

Show one eval report where a change failed. A caught regression is stronger proof of engineering maturity than a demo where everything mysteriously works.

Build Assignment

Define a release gate for your Product Triage Assistant.

  • Choose ten golden cases.
  • Mark three cases as must-pass.
  • Define one schema check, one quality check, and one safety check.
  • Write the rule that decides ship, block, or needs review.

Ask the agent to critique whether the gate protects real product risk or just easy-to-score behavior.
