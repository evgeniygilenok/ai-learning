Lesson 0009 - 30 minutes

If you cannot trace an AI run, you cannot debug it.

AI product failures are usually cross-boundary failures: prompt, retrieval, model choice, tool call, schema validation, latency, or cost.

The One Idea

A trace is the story of one run. It captures the inputs, model calls, retrieved evidence, tool calls, outputs, validation results, latency, and usage.

Tracing turns "the model was bad" into a debuggable question: which boundary failed, and can we reproduce it?

Run Trace Shape

{
  "runId": "run_2026_06_001",
  "userTask": "Triage enterprise refund request",
  "spans": [
    { "name": "retrieve", "latencyMs": 84, "chunkIds": ["refunds:enterprise:30-day"] },
    { "name": "model", "model": "primary-small", "inputTokens": 1320, "outputTokens": 210 },
    { "name": "validate", "schema": "ProductBriefV1", "status": "passed" }
  ],
  "result": "completed",
  "costBucket": "low"
}

What A Trace Lets You Ask

  • Was the wrong evidence retrieved?
  • Did the model ignore a rule or lack the rule?
  • Did schema validation catch a malformed output?
  • Was latency dominated by retrieval, model generation, or a tool?
  • Did the request exceed a product cost budget?

Signals To Capture First

SignalWhy It MattersDo Not Log
Prompt version Connects behavior changes to deployed prompt changes. Secrets or raw private data without a retention policy.
Retrieved chunk IDs Shows whether the answer had the right evidence available. Full restricted documents unless access and retention are designed.
Validation status Separates model generation from application acceptance. User-sensitive fields that are not needed for debugging.
Latency and token usage Turns speed and cost into product budgets. API keys, access tokens, or hidden system secrets.

Interactive Drill: Which Span Is Most Useful?

The final answer cites a policy that does not exist in the retrieved evidence.

The UI refuses to render the brief because riskLevel is missing.

Median response time doubled after switching to a larger model.

Build Assignment

Add a trace design to the Product Triage Assistant.

  • Name the spans for input validation, retrieval, model call, output validation, and tool execution.
  • Choose five fields you will log for every run.
  • Choose three fields you will intentionally avoid logging.
  • Define one latency budget and one cost budget in product terms.

Ask the agent to review your trace schema for debuggability and privacy risk.
