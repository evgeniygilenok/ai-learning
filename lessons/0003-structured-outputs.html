Lesson 0003 - 30 minutes

Structured outputs turn model text into product data.

A chatbot can return prose. A product feature usually needs typed fields, validation, and predictable failure behavior.

The One Idea

Structured output is the difference between "the model said something useful" and "the application received data it can safely route, render, store, or test."

For your first portfolio app, structured output lets a product triage assistant create a brief your UI can display consistently and your evals can inspect field by field.

Product Brief Schema

{
  "type": "object",
  "additionalProperties": false,
  "required": [
    "summary",
    "assumptions",
    "riskLevel",
    "risks",
    "nextQuestion",
    "suggestedNextAction"
  ],
  "properties": {
    "summary": { "type": "string" },
    "assumptions": {
      "type": "array",
      "items": { "type": "string" }
    },
    "riskLevel": {
      "type": "string",
      "enum": ["low", "medium", "high"]
    },
    "risks": {
      "type": "array",
      "items": { "type": "string" }
    },
    "nextQuestion": { "type": "string" },
    "suggestedNextAction": { "type": "string" }
  }
}

Why Each Field Exists

summary: UI preview assumptions: honesty riskLevel: routing risks: review nextQuestion: uncertainty suggestedNextAction: workflow

The schema is not just formatting. It encodes what the product needs from the model. If a field does not drive UI, workflow, evals, or storage, it may not belong.

Good Schema Versus Fragile Schema

ChoiceWeakBetterWhy
Risk "risk": "seems bad" "riskLevel": "low | medium | high" Enums make routing and tests easier.
Uncertainty No place for doubt. assumptions and nextQuestion The model can say what it does not know instead of inventing.
Extensibility One giant markdown blob. Small fields with single jobs. Fields can be rendered, filtered, stored, and evaluated independently.

Mini Quiz: Should This Be A Field?

"riskLevel" will decide whether the brief needs engineering review.

"delightfulTone" is interesting, but no UI, workflow, storage, or eval uses it.

"nextQuestion" forces the model to expose missing information.

Build Assignment

Design the v1 schema for your Product Triage Assistant. Keep it small. Aim for six fields or fewer.

  • Every field must have a product reason.
  • Use enums when the app needs routing decisions.
  • Use arrays for repeatable bullets.
  • Include one field that handles uncertainty.

After you draft it, ask the agent to review it like a senior engineer: "Which fields are unnecessary, ambiguous, or hard to evaluate?"
