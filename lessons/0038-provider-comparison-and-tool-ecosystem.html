Lesson 0038 - 45 minutes

Compare providers on one task, not on vibes.

Provider literacy means you can ship with one default stack while still understanding how model, tool, and connector choices change product behavior.

The One Idea

Do not learn every provider as a separate curriculum. Run the same small task through a few providers and compare the contract that matters: quality, latency, cost, structured output reliability, tool behavior, safety controls, and developer ergonomics.

Quality

Did the answer meet the task-specific rubric, not just sound smart?

Contract reliability

Did structured outputs, schemas, tool calls, and refusal behavior stay stable?

Latency and cost

Can the provider meet the product budget under realistic request volume?

Tool ecosystem

How do function tools, hosted tools, MCP servers, and connectors change control ownership?

Observability

Can you trace the model call, tool call, retrieval, error, and approval path?

Exit cost

How painful would it be to switch providers for this use case?

MCP Boundary

MCP standardizes how AI apps connect to external data sources and tools. That is useful, but it does not remove your application security work.

  • Filter tools before exposure.
  • Require approval for sensitive operations.
  • Trace tool discovery and execution.
  • Treat connector outputs and tool metadata as untrusted context.

Interactive Drill: Switch Or Stay?

A new model is slightly cheaper but breaks 4 of 20 structured outputs.

A provider cuts p95 latency in half with equal eval scores and stable schemas.

An MCP server exposes ten write tools when the feature only needs read-only search.

Build Assignment

Create docs/provider-comparison.md.

  • Pick one task from your portfolio.
  • Run it through OpenAI, Anthropic, Gemini, and one local or open model if practical.
  • Score quality, latency, cost, schema reliability, tool behavior, observability, and exit cost.
  • Write one recommendation and one switching trigger.
  • If you test MCP or connector-style tools, document tool filtering and approval policy.

Ask the agent to challenge your recommendation using eval evidence, not provider preferences.
