Lesson 0026 - 35 minutes

Design an AI support assistant around trust, escalation, and source control.

Support is a perfect system-design practice case because wrong answers can frustrate users or create policy risk.

The One Idea

An AI support assistant should answer from approved knowledge, cite policy, detect when it lacks evidence, and escalate to humans when the issue is high-risk or account-specific.

Inputs

User message, account tier, locale, entitlement, product area, and conversation history.

Context

Approved help docs, product status, account-safe facts, and policy snippets with citations.

Actions

Draft reply, ask clarifying question, create ticket, escalate, or suggest safe self-service steps.

Quality

Groundedness, deflection quality, escalation accuracy, CSAT signals, latency, and containment risk.

Design Skeleton

message -> classify intent/risk
  -> retrieve approved docs
  -> answer with citations
  -> if account-specific or risky:
       create human-review ticket
  -> trace, score, and learn from feedback

Non-Negotiables

  • Do not invent policy.
  • Do not expose cross-account data.
  • Escalate billing, legal, security, and outage ambiguity.
  • Keep user-facing uncertainty explicit.

Interactive Drill: Answer Or Escalate?

User asks where to find invoice downloads; docs contain exact steps.

User asks for a refund exception outside published policy.

User reports possible account compromise.

Build Assignment

Whiteboard an AI support assistant.

  • Draw the input, retrieval, answer, escalation, and feedback flow.
  • Define three escalation rules.
  • Choose three eval cases and two metrics.
  • List one privacy and one hallucination mitigation.

Ask the agent to interview you on this design for ten minutes.
