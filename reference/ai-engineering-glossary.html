AI Engineering Glossary

A practical reference for building AI products. Each term is framed as an engineering decision, not as trivia.

Runtime

LLM

A large language model predicts useful text or structured data from context. In product code, treat it as a probabilistic service behind clear input and output contracts.

Runtime

Token

A chunk of text the model reads or writes. Tokens drive context length, latency, and cost, so token budgets are product constraints.

Runtime

Context Window

The maximum amount of information the model can consider in one request. AI engineers decide what deserves to fit inside it.

Contract

Prompt

The instruction and context sent to the model. A strong prompt names the task, boundaries, examples, output format, and failure behavior.

Contract

Structured Output

A model response constrained to a schema such as JSON. This is what lets AI behavior feed normal application code without fragile parsing.

Contract

Tool Calling

The model requests an external action through a declared function schema. Your application still owns execution, permissions, retries, and audit logs.

Search

Embedding

A vector representation of content used for similarity search, clustering, and retrieval. Embeddings are the bridge between language and search infrastructure.

Search

RAG

Retrieval-augmented generation: retrieve relevant external information, put it into context, then ask the model to answer grounded in that context.

Search

Reranking

A second pass that reorders retrieved chunks for relevance. It is often used after a broad vector or hybrid search pass.

Quality

Eval

A repeatable test of AI behavior. Evals can use expected answers, human labels, code-based checks, or model-based judges.

Quality

Trace

A record of the model calls, prompts, retrieved context, tool calls, outputs, costs, and latency from a run. Traces make failures debuggable.

Risk

Hallucination

A plausible but unsupported output. Good systems reduce this with grounding, citations, uncertainty behavior, and evals.

Risk

Prompt Injection

An attempt to make the model ignore intended instructions or misuse tools. It can come directly from a user or indirectly through retrieved content.

Risk

Excessive Agency

A system gives a model more power than the task requires. The fix is least privilege, scoped tools, confirmations, and auditability.

Workflow

Agent

A system where the model can choose steps, call tools, use state, and continue over multiple turns or tasks. Useful agents need boundaries.
