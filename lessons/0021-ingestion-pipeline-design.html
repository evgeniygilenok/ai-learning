Lesson 0021 - 30 minutes

RAG quality begins before search: in the ingestion pipeline.

A retriever can only find evidence your pipeline parsed, chunked, enriched, embedded, and indexed correctly.

The One Idea

Ingestion is the supply chain for RAG. It turns messy source systems into clean, permission-aware chunks that can be retrieved, cited, refreshed, and deleted.

A production-shaped pipeline records provenance at every step so bad answers can be traced back to bad sources, bad parsing, or bad chunking.

Extract

Read source content, attachments, titles, authors, timestamps, and access metadata.

Normalize

Convert content into stable text, remove boilerplate, preserve headings, and detect document type.

Chunk And Enrich

Split by meaning, add source IDs, section paths, permissions, freshness, and domain tags.

Embed And Index

Embed only changed chunks, store vectors, and make deletion or re-indexing repeatable.

Pipeline Record

type IngestionRun = {
  sourceId: string;
  tenantId: string;
  sourceVersion: string;
  parserVersion: string;
  chunkerVersion: string;
  chunksCreated: number;
  chunksDeleted: number;
  status: "complete" | "failed";
};

Design Rule

Do not index anonymous text. Every chunk should know where it came from, who can see it, how fresh it is, and how to remove it later.

Interactive Drill: Ingestion Risk

A pipeline embeds chunks without source IDs.

A pipeline stores parser version and chunker version for each run.

A deleted source document leaves its old vectors in the index.

Build Assignment

Design the ingestion pipeline for the Enterprise Knowledge Assistant.

  • Name three source systems.
  • Define source, chunk, and ingestion-run records.
  • Choose how changed, deleted, and failed documents are handled.
  • Add one metric for freshness and one metric for parsing failures.

Ask the agent to review whether your chunks are traceable, refreshable, and deletable.
