Lesson 0031 - 35 minutes

Orchestration decides who controls the next step.

Before choosing frameworks, decide the control pattern: one workflow, a manager with bounded helpers, a handoff to a specialist, or a background job.

The One Idea

Agent orchestration is not "let the model figure it out." It is the application-level decision about which component owns the run, which tools are available, when control can move, and who writes the final user-facing answer.

For production AI engineering, start with the narrowest orchestration that solves the job. Add specialists, handoffs, and queues only when the workflow needs different instructions, permissions, latency behavior, or ownership.

Explicit Workflow

Use when the steps are known: intake, clarify, plan, dry run, review, execute. Product code owns transitions.

Manager With Tools

Use when one agent should keep ownership while calling specialist capabilities, such as summarize, search, or classify.

Handoff

Use when a specialist should take over the next response because it has distinct policy, tone, tools, or domain authority.

Background Run

Use when the workflow is long, retry-heavy, or not worth keeping inside a request/response path.

Decision Contract

type OrchestrationDecision = {
  pattern: "workflow" | "manager_tools" | "handoff" | "background";
  owner: "app" | "manager_agent" | "specialist_agent" | "worker";
  allowedTools: string[];
  nextState: string;
  finalAnswerOwner: string;
  approvalRequired: boolean;
};

Design Questions

  • Who owns the final answer right now?
  • Which tools are available in this branch?
  • What state must move across the boundary?
  • Can the user inspect or approve the transition?
  • What stops the workflow from looping?

Interactive Drill: Pick The Pattern

A coding assistant must read a ticket, ask one clarification, draft a plan, wait for approval, then produce a PR checklist.

A research assistant should answer users directly but can ask a summarizer and a citation checker for help.

A general support agent detects a refund dispute that must follow separate billing policy and tooling.

Senior Interview Angle

When asked to design an agent, start by naming the control owner. This makes autonomy, permissions, state, latency, and evaluation easier to reason about.

Build Assignment

For a Jira-style coding workflow agent, write an orchestration decision table with four rows:

  • Normal small ticket.
  • Ambiguous ticket missing acceptance criteria.
  • Security-sensitive change touching authentication.
  • Long-running dependency analysis over a large repository.

Ask the agent to challenge whether each row has the narrowest orchestration pattern that can work.
