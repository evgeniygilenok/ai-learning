# Mission: AI Engineering

## Why
Become an AI Engineer / AI Product Engineer by using an existing senior software engineering background to ship reliable AI features, not by detouring into research-heavy ML. The practical target is to build portfolio-grade systems that can survive senior interviews and eventually fintech-style production constraints.

## Success looks like
- Build a streaming LLM app with structured outputs, logging, and basic cost awareness.
- Build a production-shaped RAG system with citations, access-control thinking, and retrieval evaluation.
- Build an agentic workflow that uses tools, preserves state, and asks for human approval before risky actions.
- Explain AI system designs clearly: latency, cost, model routing, observability, evaluation, and security.
- Present a small portfolio that proves product judgment, engineering judgment, and AI-specific reliability work.

## Constraints
- Use TypeScript and existing web/product engineering strengths as the default path.
- Keep math practical at first; learn deeper ML theory only when it improves implementation decisions.
- Prefer short lessons with tangible artifacts over long passive study blocks.
- Treat security and evaluation as ongoing habits, not late-stage polish.

## Out of scope
- Training foundation models from scratch.
- Becoming a research scientist as the first goal.
- Deep GPU infrastructure before the product engineering path is solid.
- Collecting every AI framework; tools should serve the project, not become the curriculum.
