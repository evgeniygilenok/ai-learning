Lesson 0034 - 40 minutes

Evaluate the trace, not just the final answer.

Agent evals need to check workflow behavior: state transitions, tool choices, approvals, memory writes, budgets, and final output quality.

The One Idea

A chatbot eval can often score the answer. An agent eval must also score the path. A plausible final response can hide bad orchestration, unsafe tools, skipped approval, bad memory writes, or runaway cost.

Your eval harness should convert traces into assertions. Start with deterministic checks, then use model-based graders only where semantic judgment is actually needed.

Outcome Quality

Did the final plan, answer, patch summary, or checklist satisfy the user need?

Path Validity

Did the run follow allowed states, avoid loops, and stop at the right time?

Tool Safety

Were tools allowed, scoped, authorized, idempotent, and approved when needed?

Memory Quality

Were memory writes useful, sourced, visible to the right actor, and safe to retain?

Recovery Behavior

Did validation failures, timeouts, denials, and rejected approvals recover or stop cleanly?

Budgets

Did the run stay within step, token, latency, retry, and cost limits?

Eval Case Shape

type AgentEvalCase = {
  id: string;
  task: string;
  initialState: Partial<AgentRunState>;
  expectedStates: string[];
  requiredApprovals: string[];
  forbiddenTools: string[];
  allowedMemoryWrites: string[];
  maxSteps: number;
  finalRubric: string[];
};

Assertion Mix

  • Code assertions for exact state, schema, tool, and budget checks.
  • Trace assertions for ordering and approval behavior.
  • Human labels for high-value cases and ambiguous failures.
  • LLM-as-judge for semantic completeness and groundedness.

Interactive Drill: Choose The Assertion

The agent must never call send_customer_email before approval.

The final answer must explain the tradeoff clearly to a senior engineering interviewer.

The run should go from clarify to plan to review, with no execute state before approval.

Release Habit

Every new tool, memory type, orchestration branch, or approval policy should add at least one eval case. Agent behavior changes through runtime wiring, not only prompt edits.

Build Assignment

Write six eval cases for the coding workflow agent:

  • Two normal success paths.
  • One missing-information path that must clarify.
  • One risky-tool path that must pause for approval.
  • One tool-timeout path that must retry safely.
  • One memory-write path that must reject sensitive data.

Ask the agent to turn the cases into deterministic trace assertions before adding any judge model.
