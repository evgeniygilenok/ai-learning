Lesson 0020 - 35 minutes

AI infrastructure is a set of tradeoffs, not a trophy case.

Use infrastructure when it solves a product constraint: reliability, data control, latency, cost, compliance, or portability.

The One Idea

Plan v2 treats AI infrastructure as optional because the first goal is becoming an AI product engineer. You should understand infrastructure without letting it swallow the portfolio.

The right question is not "Should I use Kubernetes, self-hosted inference, or an AI gateway?" It is "What constraint do I have that simpler product architecture cannot meet?"

Managed API

Best default for fast product iteration, strong model quality, and minimal operational load.

AI Gateway

Useful for provider routing, rate limits, audit policy, cost controls, and centralized observability.

Self-Hosted Inference

Consider for data control, specialized models, predictable high volume, or hard provider constraints.

Container Platform

Useful when deployment, scaling, background workers, and service boundaries need reproducibility.

Decision Record Shape

type InfraDecision = {
  constraint: "latency" | "cost" | "data_control" | "reliability" | "compliance";
  defaultChoice: "managed_api";
  considered: ["gateway", "self_hosted", "containers"];
  decision: string;
  tradeoffs: string[];
  revisitWhen: string;
};

Questions To Ask

  • What latency and cost budget are we missing?
  • What data cannot leave our control boundary?
  • What reliability problem does this infrastructure solve?
  • Who will operate and debug it?
  • What simpler option did we reject, and why?

Interactive Drill: Which Infrastructure Choice?

You are building a portfolio demo with low traffic and need the best model quickly.

A company needs central rate limits, provider routing, usage budgets, and model audit logs.

A regulated tenant requires a specialized model and strict data-control boundary.

Portfolio Habit

It is fine to say "I would not use Kubernetes here." Senior judgment includes declining infrastructure that adds operational burden before the product needs it.

Build Assignment

Write an infrastructure decision record for your portfolio system.

  • State the default deployment choice for v1.
  • Name one reason to add an AI gateway later.
  • Name one condition that would justify self-hosted inference.
  • Name one infrastructure choice you are intentionally postponing.

Ask the agent to challenge whether each infrastructure choice follows from a real constraint.
