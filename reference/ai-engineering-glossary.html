AI Engineering Glossary

A practical reference for building AI products. Each term is framed as an engineering decision, not as trivia.

Runtime

LLM

A large language model predicts useful text or structured data from context. In product code, treat it as a probabilistic service behind clear input and output contracts.

Runtime

Token

A chunk of text the model reads or writes. Tokens drive context length, latency, and cost, so token budgets are product constraints.

Runtime

Context Window

The maximum amount of information the model can consider in one request. AI engineers decide what deserves to fit inside it.

Contract

Prompt

The instruction and context sent to the model. A strong prompt names the task, boundaries, examples, output format, and failure behavior.

Contract

Structured Output

A model response constrained to a schema such as JSON. This is what lets AI behavior feed normal application code without fragile parsing.

Contract

Tool Calling

The model requests an external action through a declared function schema. Your application still owns execution, permissions, retries, and audit logs.

Search

Chunk

A meaningful unit of source content prepared for retrieval. Good chunks are precise enough to search and complete enough to support an answer.

Search

Embedding

A vector representation of content used for similarity search, clustering, and retrieval. Embeddings are the bridge between language and search infrastructure.

Search

Vector Search

Search that compares embedding vectors by similarity. It is useful for finding meaning matches that do not share exact keywords.

Search

Hybrid Search

A retrieval strategy that combines semantic vector search with keyword or metadata search. It often improves results for exact names, IDs, and domain terms.

Search

RAG

Retrieval-augmented generation: retrieve relevant external information, put it into context, then ask the model to answer grounded in that context.

Search

Reranking

A second pass that reorders retrieved chunks for relevance. It is often used after a broad vector or hybrid search pass.

Quality

Citation

A pointer from an answer claim back to source evidence. Citations make RAG answers inspectable and easier to debug.

Quality

Eval

A repeatable test of AI behavior. Evals can use expected answers, human labels, code-based checks, or model-based judges.

Quality

Golden Dataset

A curated set of representative test cases with expected behavior. It is the regression suite for prompts, retrieval, tools, and workflows.

Quality

Eval Gate

A release rule that decides whether an AI change can ship based on eval scores, must-pass cases, safety checks, and review thresholds.

Quality

LLM-As-Judge

An evaluation pattern where a model scores semantic behavior such as groundedness, usefulness, completeness, or policy adherence.

Quality

Groundedness

The degree to which an answer is supported by supplied evidence. A grounded answer should not add claims beyond its sources.

Quality

Trace

A record of the model calls, prompts, retrieved context, tool calls, outputs, costs, and latency from a run. Traces make failures debuggable.

Quality

Latency Budget

A product constraint for how long a user should wait. It helps decide when to stream, cache, route models, or move work to the background.

Quality

Cost Budget

A product constraint for how much an AI run, user, tenant, or workflow may spend. It turns model usage into an engineering tradeoff.

Quality

Model Routing

The policy for choosing which model or path handles a request. Good routing considers task type, risk, latency, cost, and fallback behavior.

Quality

Prompt Caching

Reuse of stable prompt or context work where supported by the runtime. It is useful only when freshness, privacy, and correctness rules are clear.

Quality

Graceful Degradation

A fallback behavior that keeps the product useful when an AI dependency is slow, unavailable, or over budget.

Quality

AI Gateway

A central layer for provider routing, rate limits, usage budgets, policy enforcement, and observability across AI calls.

Quality

Self-Hosted Inference

Running model inference inside infrastructure you operate. It can help with data control or specialized models, but adds operational cost.

Risk

Hallucination

A plausible but unsupported output. Good systems reduce this with grounding, citations, uncertainty behavior, and evals.

Risk

Prompt Injection

An attempt to make the model ignore intended instructions or misuse tools. It can come directly from a user or indirectly through retrieved content.

Risk

RAG Poisoning

A retrieval security failure where manipulated or untrusted source content is retrieved and treated as evidence.

Risk

Insecure Output Handling

Rendering, executing, sending, or storing model output without appropriate validation, escaping, approval, or policy checks.

Risk

Threat Model

A structured map of assets, attackers, boundaries, likely failures, mitigations, and signals that would reveal regressions.

Risk

Redaction

Removing or masking sensitive data before it is stored, logged, used in evals, or shown in support and analytics tools.

Risk

Excessive Agency

A system gives a model more power than the task requires. The fix is least privilege, scoped tools, confirmations, and auditability.

Workflow

Agent

A system where the model can choose steps, call tools, use state, and continue over multiple turns or tasks. Useful agents need boundaries.

Workflow

Workflow State

A named step in a controlled agent process, such as intake, clarify, plan, review, or execute. State makes autonomy inspectable.

Workflow

Human-In-The-Loop

A workflow pattern where a person reviews or approves important decisions before execution. It is essential for risky tools and external side effects.

Workflow

Memory

Persisted state the system can use later. Strong memory design defines purpose, source, visibility, retention, and deletion behavior.

Workflow

Tenant Isolation

A multi-tenant architecture property that prevents one tenant's data from entering another tenant's prompts, retrieval results, tools, memory, traces, or caches.

Workflow

Data Retention

The policy for how long prompts, outputs, traces, memories, eval data, and tool results are stored before deletion or archival.

Workflow

Background Job

Durable asynchronous work outside the request path, such as ingestion, embedding, evals, cleanup, retries, or long-running agent steps.

Workflow

Idempotency Key

A stable key that lets a retried job or tool call avoid duplicate side effects. It is essential for safe retries.
