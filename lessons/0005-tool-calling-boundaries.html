Lesson 0005 - 30 minutes

Tool calling lets the model ask. Your application decides.

The model should not directly perform privileged actions. It proposes tool calls through schemas, and product code enforces policy.

The One Idea

Tool calling gives a model a vocabulary for requesting external work: search docs, fetch a ticket, create a draft, send a message, or update a record. But a tool request is not permission.

The AI engineer owns the boundary: available tools, argument schemas, authorization, dry runs, confirmations, execution, retries, and audit logs.

Tool Schema

Name, description, and argument shape. This tells the model what it may request.

Policy Gate

Rules that decide whether a requested tool call is allowed, blocked, or requires user approval.

Executor

Normal application code that performs the action, handles failures, and returns a tool result.

Audit Trail

A durable record of request, approval, execution result, actor, timestamp, and affected resource.

Example Tool

{
  "name": "create_draft_ticket",
  "description": "Create a draft ticket for human review. Does not publish.",
  "parameters": {
    "type": "object",
    "additionalProperties": false,
    "required": ["title", "body", "riskLevel"],
    "properties": {
      "title": { "type": "string" },
      "body": { "type": "string" },
      "riskLevel": {
        "type": "string",
        "enum": ["low", "medium", "high"]
      }
    }
  }
}

Why This Is Safer

  • The tool creates a draft, not a published ticket.
  • The schema restricts arguments.
  • The app can require review for medium or high risk.
  • The executor can attach provenance and logs.

Interactive Decision: Allow, Block, Or Ask?

The model wants to create a medium-risk draft ticket from the product brief.

The model wants to email customers directly with a migration notice it just wrote.

The model wants to search read-only internal docs for "refund policy".

Security Habit

Prompt injection often becomes dangerous through tools. A malicious instruction in retrieved content is only text until your system lets it trigger an action. Tool permissions are where harmless text can become operational risk.

Build Assignment

Add one safe tool to the Product Triage Assistant design:

  • search_docs(query) for read-only lookup, or
  • create_draft_ticket(title, body, riskLevel) for a dry-run workflow.

For the chosen tool, write three policy rules: one allow rule, one ask-for-approval rule, and one block rule.
