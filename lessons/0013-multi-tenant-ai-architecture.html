Lesson 0013 - 35 minutes

Multi-tenant AI design starts before the model call.

Tenant isolation must apply to prompts, retrieval, tools, memory, traces, caches, and eval data.

The One Idea

A multi-tenant AI app has more places to leak data than a normal CRUD app because the model context is assembled from many systems.

The product boundary is: no tenant should influence, retrieve, cache, observe, or receive another tenant's data unless the product explicitly allows it.

Request Boundary

Every run carries tenant ID, user ID, role, entitlement, and data-region context.

Retrieval Boundary

Search filters apply tenant and permissions before chunks enter the context packet.

Tool Boundary

Tool executors re-check tenant authorization instead of trusting model-proposed arguments.

Observability Boundary

Logs, traces, evals, and caches avoid cross-tenant visibility and respect retention rules.

Run Context Shape

type AiRunContext = {
  tenantId: string;
  userId: string;
  roles: string[];
  entitlements: string[];
  region: "us" | "ca" | "eu";
  requestId: string;
  dataPolicy: {
    canUseTenantDocs: boolean;
    canStoreTrace: boolean;
    retentionDays: number;
  };
};

Design Rule

Authorization should happen in normal application code at every boundary. The model may summarize allowed evidence, but it should never decide which tenant data the user is allowed to see.

For interview design, say where tenant ID is enforced: API layer, retrieval filters, tool executors, storage keys, trace viewers, cache keys, and background jobs.

Interactive Drill: Is This Tenant-Safe?

The vector query includes tenantId and role filters before returning chunk text.

The cache key is only the natural-language question: "What is our refund policy?"

The model gets all matching chunks and is instructed to ignore chunks from other tenants.

Security Habit

A prompt is not an access-control mechanism. Filter, authorize, and isolate before text reaches the model.

Build Assignment

Add tenant isolation to your Enterprise Knowledge Assistant design.

  • Define tenant-aware keys for chunks, traces, memory, and caches.
  • Write one retrieval filter rule and one tool authorization rule.
  • Decide which trace fields support debugging without exposing private content.
  • Describe how a background re-index job preserves tenant boundaries.

Ask the agent to attack your design as a cross-tenant data leak review.
