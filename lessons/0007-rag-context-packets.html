Lesson 0007 - 30 minutes

RAG is the art of sending the model the right evidence packet.

Retrieval-augmented generation works when search, filtering, context assembly, and citation rules act as one product contract.

The One Idea

RAG does not make a model magically know your documents. Your application retrieves candidate evidence, builds a context packet, and asks the model to answer only from that packet.

A strong context packet is relevant, permission-safe, concise, cited, and explicit about what the model should do when evidence is missing.

Retrieved Evidence

The smallest useful chunks, ranked by relevance and filtered by permissions.

Source Handles

IDs, titles, URLs, and section paths that the final answer can cite.

Answer Rules

Instructions to use only provided evidence, cite claims, and say when the packet is insufficient.

Budget

A token budget that reserves space for the user's question and the final answer.

Context Packet Shape

{
  "question": "Can enterprise customers get a refund?",
  "evidence": [
    {
      "id": "refunds:enterprise:30-day",
      "title": "Enterprise refund policy",
      "sectionPath": ["Billing", "Refunds"],
      "text": "Enterprise customers can request a refund within 30 days if onboarding has not started."
    }
  ],
  "rules": [
    "Answer only from evidence.",
    "Cite every policy claim by evidence id.",
    "If evidence is missing, ask a follow-up question."
  ]
}

Good Answer Behavior

The answer should be narrow: "Yes, if the request is within 30 days and onboarding has not started."

It should cite the source handle and avoid adding policy details that were not retrieved.

If the user asks about a region, customer tier, or exception that is absent from the packet, the system should say it lacks evidence.

Interactive Decision: What Should Go In The Packet?

A top-ranked chunk that directly answers the question and includes source metadata.

A retrieved chunk from a restricted document the current user cannot access.

Ten loosely related chunks that fill the whole context window.

Security Habit

Filter by authorization before generation. Do not rely on the model to ignore evidence the user should not see.

Build Assignment

Design the context packet for three questions from Lesson 0006.

  • List the retrieved chunk IDs in ranked order.
  • Add a source title and section path for each chunk.
  • Write one refusal rule for missing or inaccessible evidence.
  • Draft a final answer that cites chunk IDs.

Ask the agent to inspect whether the answer says more than the evidence supports.
