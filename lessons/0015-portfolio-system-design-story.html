Lesson 0015 - 35 minutes

Your portfolio needs a system design story, not just a demo.

The plan-v2 portfolio is one cumulative system presented in slices: product UX, enterprise RAG, and controlled workflow agents.

The One Idea

A strong AI engineering portfolio proves judgment. The feature should show how you handle probabilistic behavior, retrieval, evals, tools, observability, cost, security, and tenant boundaries.

The artifact is not only code. It is the story of tradeoffs you can defend in a senior interview.

Slice 1: Product Triage Assistant

Shows model calls, streaming UX, structured outputs, validation, and product-specific failure behavior.

Slice 2: Enterprise Knowledge Assistant

Shows ingestion, chunks, retrieval, citations, tenant filters, retrieval evals, and grounded answers.

Slice 3: Controlled Workflow Agent

Shows state, tools, memory, human review, retries, traces, and approval before side effects.

Slice 4: System Design Readout

Shows architecture, latency budgets, cost controls, security model, graceful degradation, and next steps.

Interview Story Shape

1. User problem and product scope
2. AI boundaries and contracts
3. Architecture and data flow
4. Quality plan and evals
5. Observability, latency, and cost
6. Security and tenant isolation
7. Tradeoffs and future work

What To Screenshot

  • A real streaming structured-output workflow.
  • A RAG answer with citations and missing-evidence behavior.
  • An eval run showing pass/fail cases.
  • A trace showing retrieval, model call, validation, and cost.
  • A human approval screen before a side-effecting tool.

Interactive Drill: Portfolio Signal Or Noise?

A README section explains why retrieval filters happen before context assembly.

A list of ten model providers appears with no explanation of routing or tradeoffs.

An eval report shows a retrieval regression caught before deployment.

Senior Signal

Hiring teams already expect you can call an API. The differentiator is whether you can explain how the system behaves when the model is wrong, slow, expensive, missing evidence, or holding a risky tool call.

Build Assignment

Create the one-page system design brief for your portfolio project.

  • Draw the data flow from user input to final answer or workflow action.
  • List the six AI product contracts: input, context, model, output, action, quality/safety.
  • Choose three metrics: one quality, one latency, and one cost metric.
  • Add three risks and the mitigation you implemented or would implement next.

Ask the agent to interview you on the brief like a senior AI system design interviewer.
